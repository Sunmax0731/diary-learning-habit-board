        ## 採用する基準

        - 入力負荷が低い。
        - 次アクションが明確。
        - ローカルまたは閉域で検証できる。
        - docs と release evidence から状態を追える。

        ## 差別化

        日記、学習、習慣タイルを日単位で確認するローカルボードに絞り、汎用ノートや大型サービスよりも閉域アルファの検証可能性を優先します。
